IIS URL Rewrite Module 發生 HTTP 500.52 錯誤

2015-10-09 11:43:00

最近使用的IIS URL Rewrite模組發生了500.52錯誤

看起來像是NTFS權限的問題，把IIS_IUSR帳號加入後，錯誤訊息又變了


在URL Rewrite的時後，把HTTP_ACCEPT_ENCODING設定空就行了


另一個方式是把IIS的動態壓縮關閉

關閉動態壓縮

應用程式伺服器的角色服務中，勾選網頁伺服器(IIS)支援
